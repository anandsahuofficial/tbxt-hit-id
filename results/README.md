# Results

Curated outputs from the multi-signal scoring pipeline.

## Top-level CSVs

| File | What it is | Rows |
|---|---|---:|
| [`top4.csv`](top4.csv) | Hackathon judges submission - the 4 ranked picks | 4 |
| [`top5to24.csv`](top5to24.csv) | Experimental program first batch - 20 additional picks | 20 |
| [`all_candidates_tiered.csv`](all_candidates_tiered.csv) | Full strict-pass set with every per-criterion flag - the audit substrate | 137 |

All three CSVs share the **same 36-column schema** so they can be
concatenated, filtered, or joined with the same column references.

## Schema

| Group | Columns | Meaning |
|---|---|---|
| **Identity** | `id`, `smiles`, `tier` | compound ID, canonical SMILES, tier (T2_SILVER / T3_BRONZE / T4_RELAXED - T1_GOLD is empty by design, see [tier_definitions](../docs/tier_definitions.md)) |
| **Boltz dual-engine** | `boltz_kd_runA_uM`, `boltz_pb_runA`, `boltz_kd_runB_uM`, `boltz_pb_runB` | Predicted Kd (µM) and prob_binder from two independent Boltz-2 runs on separate compute backends |
| **GNINA** | `v1_gnina_kd_uM` | GNINA-CNN-predicted Kd (µM); blank for picks not in the top-30 GNINA re-rank |
| **Catalog (muni.bio onepot)** | `muni_price_usd`, `muni_chem_risk`, `muni_supplier_risk` | onepot.ai catalog price + chemistry / supplier risk strings |
| **Descriptors** | `MW_Da`, `LogP`, `HBD`, `HBA`, `HA`, `rings`, `fused_rings` | RDKit Crippen + ring counts |
| **Solubility** | `esol_logS` | ESOL-predicted log(S) where S in mol/L |
| **Novelty** | `naar_max_tanimoto` | Maximum Tanimoto to the Naar SPR-measured TBXT-binder set |
| **Reactive flags** | `forbidden_motifs` | Comma-separated list of any matched forbidden motifs (`-` = none) |
| **Strict-gate flags (✓ = pass)** | `C1_onepot_100`, `C2_non_covalent`, `C3a_MW_le_600`, `C3b_LogP_le_6`, `C3c_HBD_le_6`, `C3d_HBA_le_12`, `C4a_HA_10_30`, `C4b_HBD_HBA_le_11`, `C4c_LogP_lt_5`, `C4d_lt_5_rings`, `C4e_le_2_fused`, `C5_no_PAINS`, `C5b_no_forbidden`, `C6_naar_tanimoto_lt_085`, `C7_soluble_logS_gt_neg5` | Per-criterion pass/fail flags from the [seven-criterion strict gate](../docs/filter_chain.md) |

The criterion sub-codes (e.g. C3a, C3b, C3c, C3d) correspond to the
four constraints inside C3 (Chordoma rule). All-`✓` rows pass every
criterion; the strict-pass set in `all_candidates_tiered.csv` is the
137 such rows.

## What's not here (and why)

- **Raw poses (PDB / SDF / DCD).** Multi-GB per pick. Available on
  request; not redistributed.
- **Receptor MD trajectories.** Same - multi-GB; the receptor
  ensemble is regenerated by the pipeline.
- **Per-vendor catalog API responses.** Vendor terms preclude
  redistribution; query muni.bio directly with the SMILES from
  `top4.csv` to reproduce.
- **ADMET raw output (Rowan).** License terms preclude
  redistribution; available on request.
